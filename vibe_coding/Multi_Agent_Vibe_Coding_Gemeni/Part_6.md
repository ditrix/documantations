Вот готовый **«Мастер-промпт»** для режима **Composer (Ctrl+I)**.

Прежде чем вставить его, убедитесь, что вы создали структуру .cursor/rules/ с файлами architect.mdc, coder.mdc и ui-specialist.mdc, как мы обсудили ранее. Это активирует ваши кастомные роли внутри Cursor.

### **Глобальный промпт для развертывания проекта**

Скопируйте и вставьте это в Composer:

«Инициализируй проект "Vibe-Blog" на стеке Next.js (App Router), Prisma и Tailwind CSS. Действуй строго по этапам, используя соответствующие роли из .cursor/rules:

1. **Role: Architect** — Создай структуру папок и файл prisma/schema.prisma. Модель Post должна включать: id, title, content, createdAt. Модель Comment должна быть привязана к Post и содержать: id, authorName, text, createdAt.  
2. **Role: Coder** — Сгенерируй API-эндпоинт POST /api/comments для сохранения комментариев в БД. Добавь базовую валидацию входящих данных (текст не пустой).  
3. **Role: UI Specialist** — Создай страницу поста app/posts/\[id\]/page.tsx с использованием Tailwind. Сверстай список комментариев и форму добавления нового комментария в стиле минимализма.  
4. **Integration** — Подготовь файл .env с SQLite и выведи в чат финальные команды для терминала (npm install, npx prisma db push).»

### ---

**Что произойдет после запуска:**

1. **Cursor проанализирует инструкции**: Благодаря файлам .mdc, он поймет, какие "фильтры" качества накладывать на каждый файл (например, проверять ошибки в API, как прописано для Кодера).  
2. **Параллельное создание**: Вы увидите, как в боковой панели появляются файлы.  
3. **Контроль зависимостей**: Агент-интегратор (базовая роль Cursor) свяжет фронтенд с созданным API.

### **Финальные шаги в терминале (когда ИИ закончит):**

После генерации файлов, выполните эти команды, чтобы оживить блог:

Bash

\# 1\. Установка зависимостей  
npm install

\# 2\. Инициализация базы данных (создаст файл dev.db)  
npx prisma db push

\# 3\. Запуск локального сервера  
npm run dev
